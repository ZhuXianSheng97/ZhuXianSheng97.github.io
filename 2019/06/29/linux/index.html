第一章
Linux 与 Window 的区别
相同点
 都是多用户操作系统，都可以由许多不同的用户来使用，都可以以组成员的方式来控制资源的访问权限
 都支持多文件系统，文件资源可以通过FTP或者其他协议与其他客户机共享；
 都支持多种网络协议，如TCP/IP
 都可以提供网络服务能力，如DHCP和DNS

第二章
• 终端和shell的概念
o 终端:“终端”的概念来源于早期计算机时代，例如在Unix中，类似于显示器可以操纵计算机主机的输入/输出设备叫做终端。经过技术演进，终端逐渐被显示器和键盘取代。
o Shell:Shell的原意是指操作系统的外壳程序，在Shell中用户可以通过Shell提供的命令程序与操作系统进行交互以完成相关工作。
• 提示符 查看帮助 安装软件命令
o 提示符:Shell命令输入提示符，简称提示符，默认情况下提示符是一个”$”符号，其后是光标.
 $ 提示符前面的信息是“用户名和路径信息”其格式如下：
<用户名>@<计算机名>:<当前目录>
o “~” 表示当前用户主目录
o 查看帮助:
 man(Manual) :用于查看外部命令手册
 help:用于查看内部命令手册
另外，部分外部命令还可以通过其自身的选项“-help”来查看帮助.
o 安装软件命令
 搜索软件: apt-cache search <软件包名>
E.G. apt-cache search gedit
 安装软件:apt-get install <软件包名>
E.G. sudo apt-get install gedit (ubuntu 下安装软件需要管理员权限)
 删除软件:apt-get remove <软件报名>
E.G. sudo apt-get remove gedit (ubuntu 下删除软件需要管理员权限)
o 列出目录内容: ls -[option] [Dir or File]
 -a 显示目录下所有子目录与文件,包括隐藏文件.
 -l采用长格式来显示文件的详细信息
E.G. ls -l /usr
• 文本编辑器
o gedit: 图形化文本编辑程序
o vim : 在Shell 中运行的基于命令行的文本编辑器.
 三种模式

 正常模式: :也叫命令模式，Vim刚启动时处于该模式下，可以输入各种命令来控制Vim；
 文本模式：在该模式下可以进行文字输入
 命令行模式： 也叫底行模式，在该模式下可以在Vim的最下面一行输入命令来控制Vim，例如文件的保存、Vim的退出等。
Note: 文本模式和命令行模式不可互换
 vim启动
E.G. vim <文件名>
 vim 关闭
命令模式下：q
 vim 保存
命令模式下：w
第三章
• 基本目录命令
o pwd(Print working directory) :显示当前所在目录的绝对路径
E.G. pwd
o mkdir :创建新目录 mkdir [-option] <目录名>
 -m (mode) 设置目录权限，方法同chmod
 -p 创建存在中间路径的目录
E.G
mkdir shilingfeng
mkdir -p shi/ling/feng
o cd (change directory) : 改变目录 cd [dir]
Note:
 cd : 进入用户主目录
 cd ~ 进入用户主目录
 cd - 返回进入此目录之前所在目录
 cd .. 返回上级目录
 cd ../ 返回进入此目录前所在的目录
o rmdir （remove directory）删除制定目录rmdir [-option] <文件名>
 -p 当子目录被删除后，若父目录为空目录，则此父目录一并删除
 -v 提示删除操作成功
E.G. rmdir -pv shi/ling/feng
Note: rmdir 只能删除空目录
